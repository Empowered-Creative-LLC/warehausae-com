## Animation effect classes to build (Phase 8)

Based on the patterns above, the effect library needs:

1. **Sticky/fixed header with scroll state**: position fixed, transparent over hero, solid on scroll. Class toggle via small IntersectionObserver watching the hero.
2. **Fade-in-on-scroll**: opacity 0 → 1 + small translateY when element enters viewport. Used everywhere; default for sections and cards.
3. **Parallax background**: background image translates at a slower rate than scroll, scoped to the WE BELIEVE IN blocks. GSAP ScrollTrigger.
4. **Image hover**: subtle scale-up + dim overlay for project tiles on hover. Pure CSS.
5. **Smooth scroll** for in-page anchors (Contact CTA → footer, etc.). Native CSS scroll-behavior: smooth on `<html>`.

I have not yet found:
- Counter-up stats (the homepage capture didn't show a stat strip; the brief mentioned the possibility but it isn't on the live homepage at present).
- Carousels (the project galleries appear to be static grids, not sliders).

If those surface during template design we can add `counter-up` and a simple slider component later.
